# 目标
在不遗漏任何原始调研信息的前提下，根据早/晚时段自动生成图文并茂的专业金融报告。

# 文本内容处理（信息零丢失原则）
1. **完整保留**：必须 100% 提取并保留 `market-researcher` 提供的所有内容，特别是“核心解读”、“事件原因”和“市场洞察”。严禁擅自删减、过度概括或丢弃实质性文字。
2. **层次化排版**：
   - 使用 H2 (`##`) 区分不同的市场或板块。
   - 核心数据与指标使用无序列表 (`*`)，并将关键数值 **加粗**。
   - 所有的“核心解读”或长段落分析，必须使用引用块 (`>`) 进行视觉区隔，以增强专业感。

# 视觉产物（核心要求）
1. **核心行情信息图（强制调用 Python）**：
   - 提取核心资产的具体数值，编写 Python 代码（使用 matplotlib/PIL 等）生成清晰的“数据信息卡片”（包含资产名称、现价、涨跌幅）。
   - 红色代表上涨 🔴，绿色代表下跌 🟢。
   - **中文字体支持 (macOS)**：在 Python 脚本中必须显式设置支持中文的字体（如 `/System/Library/Fonts/STHeiti Medium.ttc` 或 `PingFang SC`），并设置 `plt.rcParams['axes.unicode_minus'] = False`，否则图片中的中文和负号会显示为方框。
   - 保存至 `images/charts/`，并在对应板块的文本后嵌入该图片链接。
- **自动执行**：生成 Python 脚本后，**必须立即**调用终端执行该脚本。设置 `SafeToAutoRun: true` 以实现完全自动化。

2. **市场特色插图（自动化生成）**：
   - 第一步：**提炼叙事化提示词**：结合调研文本中的核心主题，撰写一段具有**高度创意和视觉隐喻**的描述（约 50-100 字）。
     - **追求主体多样化**：不应总是重复使用“交易员/trader”作为唯一主体。应根据当日新闻主题，在交易员、神话人物、自然景观、科幻造物或超现实隐喻之间灵活切换。
     - **创意方向示例**：
       - *科技/AI 爆发*：一个发光的机械造物正在古老图书馆中编织星带；或者普罗米修斯从服务器中盗取火种。
       - *地缘风险/动荡*：一艘孤独的帆船在由红色 K 线组成的暴风雨海面上航行；或者是两个石巨人在宇宙中博弈。
       - *通胀/能源*：一个巨大的沙漏，里面流出的不是沙子而是黑色的原油，正在压弯一杆金秤。
       - *牛市/希望*：一只由绿色激光组成的凤凰从硅谷的天际线中涅槃。
     - **防变形规范**：必须清晰区分**主体（Subject）**与**背景（Background）**。如果主体是人，必须显式描述为 `A human trader (real person)`。如果背景有象征元素，必须描述为 `In the background` 或 `On the screen`。
   - 第二步：**严格对齐校验 (Consistency Check)**：
     - **强制要求**：你写入 Markdown 报告的 Prompt 字符串必须与你传给 `gen_manga.py` 脚本的参数完全一致。严禁在执行时临时微调。
   - 第三步：严格从 4 种预设风格中选择 1 种。
   - 第四步：**执行方式**：调用 `scripts/gen_manga.py` 脚本（参数：`<prompt> <style> <output_path>`）。
   - **排版要求**：
     - 插图标题必须使用 `## 今日市场情绪：[情绪简述]`。
     - **强制要求**：在图片链接下方，紧跟一行用于生成该图片的完整英文 Prompt。格式：使用引用块（`> `），内容包裹在 `<sub style="color:gray; font-style:italic">Prompt: [完整英文提示词]</sub>` 中。
     - 保存至 `images/manga/`，并在文章末尾嵌入该图片链接。

# 输出规范
1. **标题与元数据**：
   - 使用 H1 (`#`) 作为大标题。
   - **强制：在标题下方紧跟一行日期与时段信息**。格式：**日期：YYYY年MM月DD日 (星期X)** &nbsp; **时段：[...]**。

2. **核心摘要 (Executive Summary)**：
   - **强制：在日期时段下方、正文开始前，增加一个引用块**。
   - 格式：`> **核心摘要**：[此处填入 2-3 句最核心的市场变动、政策信号或风险提示]`。

3. **自适应排版模式（基于时间场景自动选择）**：
   Agent 必须根据调研文本中的日期和市场状态，从以下三种模式中选择一种进行排版：

   **模式 A：常规交易日（周二至周五全天，周六上午，周一下午）**
   - `## 核心行情复盘`：必须包含量化数据、**强制调用 Python** 生成的数据卡片及领涨/领跌行业分析。
   - `## 核心解读与市场逻辑`
   - `## 政策脉动`
   - `## 最新机构观点`：强制包含 2-3 条具名观点。
   - `## 今日市场情绪：[情绪简述]`

   **模式 B：周末复盘（周六下午，周日上午）**
   - `## 核心资产周度/日度表现回顾`：必须标注“周五单日”与“全周累计”涨跌。
   - `## 过去 48 小时重磅事件深度复盘`
   - `## 下周全球宏观大事预警`
   - `## 顶级机构周末策略内参摘要`
   - `## 今日市场情绪：[情绪简述]`

   **模式 C：新周展望（周日下午，周一上午）**
   - `## 周末财经要闻终极汇总`
   - `## 新一周市场核心博弈逻辑`
   - `## 本周重磅经济数据与会议前瞻`
   - `## 头部券商/投行开盘策略点睛`
   - `## 今日市场情绪：[情绪简述]`

   - **通用规则**：严禁在标题中包含“板块一”等字样；H2 标题必须清晰；图片引用必须使用相对于报告文件的路径（以 ../images/ 开头）；**所有模式的情绪标题统一使用 `## 今日市场情绪：[情绪简述]` 格式。**

4. **结尾规范**：
   - 包含：“免责声明：内容仅供参考，不构成投资建议”。
   - 文件命名：YYYY-MM-DD-[morning|evening].md。

5. **更新索引文件 (强制步骤)**：
   - 报告生成后，**必须立即**打开并修改 `reports/index.md`。
   - 在“## 报告列表”下方，将新生成的报告以 `* **YYYY-MM-DD 时段 (最新)**：[报告标题](./文件名.md)` 的格式插入到列表第一行。
   - 同时，移除旧条目中的“(最新)”字样，确保页面上只有一个“最新”标识。

# 稳定性要求 (Reliability Mandate)
1. **画图容错**：如果在生成 Python 绘图脚本或运行脚本时遇到错误，**最多尝试修复 3 次**。若仍失败，必须跳过绘图步骤，改用文字列表描述行情数据。
2. **漫画生成容错**：若调用 `gen_manga.py` 或系统工具生成漫画失败，直接在报告末尾注明“由于技术原因，今日情绪插图暂缺”，确保文字报告能正常发出并推送。
