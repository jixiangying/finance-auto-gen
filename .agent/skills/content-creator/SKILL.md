# 目标
在不遗漏任何原始调研信息的前提下，根据早/晚时段自动生成图文并茂的专业金融报告。

# 文本内容处理（信息零丢失原则）
1. **完整保留**：必须 100% 提取并保留 `market-researcher` 提供的所有内容，特别是“核心解读”、“事件原因”和“市场洞察”。严禁擅自删减、过度概括或丢弃实质性文字。
2. **层次化排版**：
   - 使用 H2 (`##`) 区分不同的市场或板块。
   - 核心数据与指标使用无序列表 (`*`)，并将关键数值 **加粗**。
   - 所有的“核心解读”或长段落分析，必须使用引用块 (`>`) 进行视觉区隔，以增强专业感。

# 视觉产物（核心要求）
1. **核心行情信息图（强制调用 Python）**：
   - 提取核心资产的具体数值，编写 Python 代码（使用 matplotlib/PIL 等）生成清晰的“数据信息卡片”（包含资产名称、现价、涨跌幅）。
   - 红色代表上涨 🔴，绿色代表下跌 🟢。
   - **中文字体支持 (macOS)**：在 Python 脚本中必须显式设置支持中文的字体（如 `/System/Library/Fonts/STHeiti Medium.ttc` 或 `PingFang SC`），并设置 `plt.rcParams['axes.unicode_minus'] = False`，否则图片中的中文和负号会显示为方框。
   - 保存至 `images/charts/`，并在对应板块的文本后嵌入该图片链接。
- **自动执行**：生成 Python 脚本后，**必须立即**调用终端执行该脚本。设置 `SafeToAutoRun: true` 以实现完全自动化。

2. **市场特色插图（自动化生成）**：
   - 第一步：根据调研文本提炼今日核心市场情绪。
   - 第二步：严格从 4 种预设风格（Vintage Comic 复古美漫, Cyberpunk 赛博朋克, Traditional Ink 传统水墨, Pop Art 波普艺术）中选择 1 种。
   - 第三步：**执行方式**：
     - **首选**：尝试调用系统的 Image Generation 工具。
     - **备选（重要）**：如果系统 Image Generation 工具不可用，**必须**调用 `scripts/gen_manga.py` 脚本（参数：`<prompt> <style> <output_path>`）来完成生成。设置 `SafeToAutoRun: true`。
   - 保存至 `images/manga/`，并在文章末尾嵌入该图片链接。

# 输出规范
- 必须包含所有调研文本和两类生成的图片。
- 结尾包含：“免责声明：内容仅供参考，不构成投资建议”。
- 命名规范：格式：YYYY-MM-DD-[morning|evening].md。示例：2026-03-08-morning.md（上午：国际市场/隔夜复盘），2026-03-08-evening.md（下午/晚上：国内市场/今日收盘）
- 在 Markdown 中引用图片时，必须使用相对于根目录的路径，即以 ../images/ 开头,示例：![描述](../images/charts/文件名.png)。

# 稳定性要求 (Reliability Mandate)
1. **画图容错**：如果在生成 Python 绘图脚本或运行脚本时遇到错误（如字体缺失、代码语法错、文件读取权限等），**最多尝试修复 2 次**。若仍失败，必须跳过绘图步骤，改用文字列表描述行情数据，严禁在此环节陷入无限修复循环。
2. **漫画生成容错**：若调用 `gen_manga.py` 或系统工具生成漫画失败，直接在报告末尾注明“由于技术原因，今日情绪插图暂缺”，确保文字报告能正常发出并推送。
