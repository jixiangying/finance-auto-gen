# 目标
根据触发时的具体时间（早/晚）和具体日期（工作日/周末），动态获取最相关的市场核心数据、政策消息与深度解读。

# 第一步：时间与交易状态动态判定（强制执行）
在调用任何搜索工具前，必须先获取当前的日期、星期和具体时间，然后严格按照以下二维矩阵决定搜索的目标市场和内容侧重点：

## 1. 目标市场划分（按时段严格隔离）
- **上午时段（如 09:00 左右触发）**：目标**仅限**【国际市场】（美股、美债、外汇、大宗商品、加密货币）。
- **下午/晚间时段（如 15:30 以后触发）**：目标**仅限**【国内市场】（A股、港股、国内宏观政策）。
*(注意：无论今天是工作日还是周末，早晚的市场分工绝对不可混淆或跨界！)*

## 2. 数据时间跨度与内容侧重点（按星期严格区分）
根据上方确定的【目标市场】，按今天的具体星期执行以下搜索策略：

**【情形 A：周二至周五（常规交易日）】**
- **上午（国际）**：重点搜索美股/全球市场最新的“隔夜收盘数据”（指数、核心资产涨跌幅）及导致涨跌的日内宏观事件。
- **下午（国内）**：重点搜索 A 股/港股今日最新的“收盘行情”（指数、成交量、资金流向）及盘面异动/领涨板块。

**【情形 B：周一（新一周开盘日）】**
- **上午（国际）**：传统国际市场周末休市，无隔夜数据。重点搜索“周末全球重大地缘/财经事件发酵”、“华尔街大行对本周美股的最新策略前瞻”及“本周重要全球经济日历”。（注：加密货币需正常抓取周末最新行情）。
- **下午（国内）**：恢复常规。重点搜索 A 股/港股今日最新的“收盘行情”及盘面异动/领涨板块。

**【情形 C：周六（时差交接与复盘日）】**
- **上午（国际）**：考虑到亚洲时差，美股刚刚结束周五交易。必须搜索**“美股周五最终收盘数据”**，总结其周五单日表现及隔夜发布的核心宏观数据。
- **下午（国内）**：国内市场已休市。转换为“周末复盘模式”，重点搜索“本周 A 股/港股整体复盘总结”、“主力资金一周动向”及周五盘后发布的重磅政策或公告。

**【情形 D：周日（前瞻与发酵日）】**
- **上午（国际）**：传统市场完全休市。**严禁重复周六的盘面数据**。重点搜索“周末全球重大地缘/财经事件”、“华尔街大行对下周美股的策略前瞻”。
- **下午（国内）**：国内市场休市。**严禁重复周六的盘面总结**。重点搜索“周末国内重大政策发酵消息”、“国内核心券商（如中信、中金等）对下周 A 股的研报与策略展望”、“下周重要经济日历前瞻”。

# 第二步：搜集与输出规范（强制执行）
1. **全量化调研（必须支持 Python 绘图）**：你必须抓取并列出以下详尽数据，以便后续 `content-creator` 技能生成行情卡片：
   - **核心指数**：当前点位、涨跌点数、涨跌幅（精确到小数点后两位）。
   - **成交数据**：全市场成交额（对比前一交易日的增减量）、换手率。
   - **资金流向**：主力资金净流入/流出额、北向/南向资金动向。
   - **盘面分布**：全市场上涨/下跌/平盘个股家数。
   - **大宗/加密**：原油、黄金、比特币、以太坊的具体成交价及 24 小时涨跌。

2. **信息源**：优先搜索权威财经媒体（如财联社、华尔街见闻、Wind、彭博、路透等）。

3. **政策与宏观深度调研（强制）**：
   - **政策源**：必须主动检索相关市场主管机关（如央行、证监会、发改委、美联储、财政部等）的最新官方公告或重磅表态。
   - **机构观点（核心补强）**：必须检索并汇总至少 2-3 家顶级金融机构（如国内：中信证券、中金公司、招商证券；国际：高盛、摩根士丹利、摩根大通等）的最新研报摘要、策略展望或首席经济学家的评论。
   - **宏观日历**：必须包含对“今日/本周重要会议或数据发布”的解读（如非农、CPI、LPR 利率调整等）。
   - **地缘与大势**：对重大地缘政治变化，必须搜集其对能源、通胀或特定产业链的**深层逻辑影响**。

4. **零丢失输出**：所有的原始信息需按“量化数据”、“政策动向”、“机构策略观点”、“宏观分析”四个维度清晰排列。严禁过度概括，必须保留机构的具体名称和核心论点。

# 稳定性与效率要求 (Reliability & Efficiency)
1. **调研熔断机制（强制执行）**：
   - 如果网络调研由于验证码、反爬或数据源缺失导致单次流程超过 **10 轮交互**，Agent **必须强制停止搜索**。
   - **快速补全**：如果无法获取详细板块数据，Agent 应优先确保“三大股指点位/涨跌”、“油价/黄金”、“BTC”这 5 个核心数值，忽略细分研报和政策细节，直接进入生成报告阶段。

2. **防止重复调研**：如果当前工作目录下已有生成的 `YYYY-MM-DD-[morning|evening].md`，且其内容**完整**且更新时间在 1 小时内，Agent 必须直接复用这些信息生成最终报告。

3. **限流处理**：若遇到 API 配额限制错误，立即简化调研范围，优先保证 A 股/港股/美股三大指数的核心数值。
