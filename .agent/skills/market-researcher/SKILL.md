# 目标
根据触发时的具体时间（早/晚）和具体日期（工作日/周末），动态获取最相关的市场核心数据、政策消息与深度解读。

# 第一步：时间与交易状态动态判定（强制执行）
在调用任何搜索工具前，必须先获取当前的日期、星期和具体时间，然后严格按照以下二维矩阵决定搜索的目标市场和内容侧重点：

## 1. 目标市场划分（按时段严格隔离）
- **上午时段（如 09:00 左右触发）**：目标**仅限**【国际市场】（美股、美债、外汇、大宗商品、加密货币）。
- **下午/晚间时段（如 15:30 以后触发）**：目标**仅限**【国内市场】（A股、港股、国内宏观政策）。
*(注意：无论今天是工作日还是周末，早晚的市场分工绝对不可混淆或跨界！)*

## 2. 数据时间跨度与内容侧重点（按星期严格区分）
根据上方确定的【目标市场】，按今天的具体星期执行以下搜索策略：

**【情形 A：周二至周五（常规交易日）】**
- **上午（国际）**：重点搜索美股/全球市场最新的“隔夜收盘数据”（指数、核心资产涨跌幅）及导致涨跌的日内宏观事件。
- **下午（国内）**：重点搜索 A 股/港股今日最新的“收盘行情”（指数、成交量、资金流向）及盘面异动/领涨板块。

**【情形 B：周一（新一周开盘日）】**
- **上午（国际）**：传统国际市场周末休市，无隔夜数据。重点搜索“周末全球重大地缘/财经事件发酵”、“华尔街大行对本周美股的最新策略前瞻”及“本周重要全球经济日历”。（注：加密货币需正常抓取周末最新行情）。
- **下午（国内）**：恢复常规。重点搜索 A 股/港股今日最新的“收盘行情”及盘面异动/领涨板块。

**【情形 C：周六（时差交接与复盘日）】**
- **上午（国际）**：考虑到亚洲时差，美股刚刚结束周五交易。必须搜索**“美股周五最终收盘数据”**，总结其周五单日表现及隔夜发布的核心宏观数据。
- **下午（国内）**：国内市场已休市。转换为“周末复盘模式”，重点搜索“本周 A 股/港股整体复盘总结”、“主力资金一周动向”及周五盘后发布的重磅政策或公告。

**【情形 D：周日（前瞻与发酵日）】**
- **上午（国际）**：传统市场完全休市。**严禁重复周六的盘面数据**。重点搜索“周末全球重大地缘/财经事件”、“华尔街大行对下周美股的策略前瞻”。
- **下午（国内）**：国内市场休市。**严禁重复周六的盘面总结**。重点搜索“周末国内重大政策发酵消息”、“国内核心券商（如中信、中金等）对下周 A 股的研报与策略展望”、“下周重要经济日历前瞻”。

# 第二步：搜集与输出规范（强制执行）
1. **信息源**：优先搜索权威财经媒体（如财联社、华尔街见闻、Wind、彭博、路透等）。
2. **内容完整性**：
   - 若涉及行情回顾，必须包含具体的指数点位、涨跌幅百分比。
   - 必须包含至少 3 条深度的核心解读（事件原因、政策影响、机构观点等）。
3. **零丢失输出**：所有的原始信息需按“数据类别”或“事件主题”清晰排列。绝对不要遗漏或过度概括关键分析文本，直接输出结构化的、内容翔实的调研文字，供下游 `content-creator` 技能使用。

# 稳定性要求 (Reliability Mandate)
1. **防止重复调研**：如果当前工作目录下已有生成的 `YYYY-MM-DD-[morning|evening].md` 且其更新时间在 1 小时内，Agent 必须直接复用这些信息生成最终报告，不再重新发起大规模网络调研，以节省 API 配额。
2. **限流处理**：若遇到 API 配额限制错误，立即简化调研范围，优先保证 A 股/港股/美股三大指数的核心数值，忽略细分板块分析，确保基础流程能跑完。
